CECS 282 - Midterm 2 review

Topics

Pointers and dynamic memory

  • Syntax for pointers: difference between & and *; between * and ->; between passing a pointer and passing an object.
  • The "new" and "delete" keywords and how they relate to putting variables on the heap
  • Facts about the heap:
    • Is the heap structured / can you predict where a value will go when placed on the heap?
    • When do heap variables get freed?
    • If a pointer on the stack gets popped, does the heap value that it points to get deleted?
    • What are memory leaks / how do they happen?
  • Philosophy:
    • why put things on the heap?
    • how to manage the lifetime of heap variables?
    • contrast variables on the stack vs. those on the heap
  • Ex: in this code, describe the difference between p and the Rational that it points to:

    Rational *p = new Rational(5, 3);
  • Creating arrays on the heap: what does this let us do? how do we delete them?

Classes and objects

  • General class design: .h vs .cpp files, scoping rules, include guards, accessors and mutators.
  • Operator overloading, particularly:
    • arithmetic operators, like operator+ and operator- (both binary and unary)
    • operator<< for output
    • operator= for assignment
  • Construction and destruction:
    • default constructor
    • copy constructor
    • destructor
  • const objects and const functions
    • What member functions can you call on a const object?
    • What is a const function?
    • What do the three "const" keywords below do?:

      const string &MyClass::SomeFunction(const Rational &r) const { ...
  • Why use the "const reference" parameter pattern?
  • Why don't we use const reference with fundamental types?
  • Deep vs shallow copies
  • The role of destructors, copy constructors, and assignment operators in managing heap-based member variables

STL classes

  • string: know the substr function; how to index with [] or .at().
  • array: contrast std::array with built-in arrays.
  • ostringstream: construct an ostringstream object and use it to do formatted string construction.

Exceptions

  • throw and catch simple exceptions (we'll always use std::out_of_range on the exam) to report and respond to simple errors

Misc lecture topics
