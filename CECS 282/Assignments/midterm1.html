CECS 282 - Exam 1 review

Topics

Primitive types

  • Know valid fundamental types and their domains
  • Know implicit conversion details:
    • which variable types can be implicitly converted?
    • if two variables of different types are in the same expression, which one gets promoted/converted into the other?
  • Practice problem: int x = 1; while (!x != 10) {cout << "Hi"; x++;} ------ how many times will this output? Ans: Buzz Lightyear.
  • rules about the sizes of various fundamental types; which types have guaranteed sizes and which are compiler-dependent

References, arrays, and pointers

  • Declare and use a reference variable
  • Declare and use pointers
  • Declare and use an array or a matrix
  • Differences between Java and C++ arrays
  • Compare and contrast pointers and references: syntax, capabilities
  • Arrays as pointers
  • C-style strings as char arrays/char pointers
  • Follow complicated function calls involving references/pointers and track how variable values change over the program's lifetime
  • Contrast array subscript syntax with pointer arithmetic syntax, e.g., myArray[5] vs. *(myArray + 5).
  • Write a for loop to iterate through an array using only pointer arithmetic and NO subscripts

General

  • Know the standard preferred way of organizing functions into multiple files
    • .h file contains function prototypes/declarations only
    • .cpp file contains implementation of functions
  • Contrast declarations with definitions 
  • Know how the #include preprocessor directives work
  • Write and work with functions/methods.

Memory management

  • Give the layout of the program stack following a sequence of statements/declarations/function calls; including local variables, function parameters, and function return type.
  • Know the stack layout rules for: fundamental types; arrays; pointers; references; functions.
  • Seriously.

Misc lecture topics

  • Basic input/output with cin/cout. Nothing more complicated than your Project 1.
  • Nothing specifically about Othello, Monty Hall, Tic-Tac-Toe, etc. But general concepts, like passing an array to a function, using cin with pointers, 2D arrays...
  • Nothing about random numbers.
  • Know the ?: operator.
